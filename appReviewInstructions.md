# App-By-App Review Instructions

1. Create an initial "App-By-App Review" document as a sibling to this document with a "statement of purpose"
2. One-by-one, review each folder (except "common" and "docs"), and add a section for that folder to the report listing the following
   1. The name of the folder
   2. The title of the app (the <title> tag of the "index.html" in that folder)
   3. Whether or not the app is referenced in the root "index.html"
   4. the most recent "date modified" of all files in that app folder or any subfolders
   5. describe the structure/stack category (e.g., “zero-build-react” vs “plain old javascript” vs “kaplay.js” vs other)
      1. This should be able to be inferred from the index.html for that app
   6. identify missing docs and test coverage gaps
   7. identify command-line/static-data utilities inside the app folder
3. Once all folders have been reviewed, create an additional summary section at the end of the review containing two lists: one of all the apps referenced in the root index.html, and one list of all those that aren't. Let's also include in that list the most recent date modified for each app, and put both lists in order by oldest first.
4. Once the report is complete, let's review "generalSpecifics_tasks.md" section 3.1 to see what points we can mark as complete.
